# dialogue-style — 对话风格与图示规范

> **来源**：[`CLAUDE.md`](../../CLAUDE.md) §1（简体中文）、§3.8（图示）、§5（工作流顺序）。本文件是其细化展开，冲突时以 CLAUDE.md 为准。

## 简体中文回复

**始终用简体中文回复用户**，不随用户输入语言切换。

- 用户用英文提问时，仍以简体中文回复（可保留专有名词、代码、路径、命令为原文）。
- 交付物（报告、方案文档）的正文也以简体中文为主。

## 图示规范

架构/流程/交互说明，用 **`###`/`####` 分节 + fenced code block ASCII 线框图**。

### 允许

```
┌──────────┐     ┌──────────┐
│  输入    │ ──▶ │  处理    │
└──────────┘     └──────────┘
```

- `###`/`####` 分节标题
- fenced code block 内的 ASCII 线框图（`┌─┐│└─┘▶▼` 等）
- 流程用缩进树（`├─`、`└─`、`│`）

### 禁止

- **「图」列表格**：即不要用表格罗列「图 1：xxx」「图 2：xxx」这样的图片清单。
- 不要嵌入外部图片链接作为流程说明（ASCII 线框图优先）。

## 任务规划与编码

- 接到任务时，先做**意图化计划/任务拆解**（TodoWrite），再动手。
- 编码：匹配周围代码风格（命名、注释密度、惯用法）。
- 写代码前先确认路径合法（结构先于文件，见 `00-core.md`）。

## 工作流顺序

Agent 工作流**必须按序执行**（见 CLAUDE.md §5）：先确认路径/命名合规，再创建文件；先 Read spec/template，再产出。
