# indexed

> AI 协作工作区基线框架——让 AI 成为你的伙伴，而非简单的编程工具。

## 这是什么

indexed 是一个**为 AI 助手量身定制的协作工作区**。它提供一套规则框架，让 Claude Code、ZCode 等 AI 助手在你的项目中拥有长期记忆、一致的行为规范、可发现的能力体系。

**不是代码库，是协作框架。** 你下载它、初始化它，然后 AI 助手就在这个框架里"知道该怎么做"——按规范放文件、遵循命名约定、使用已有工具、不越界行事。

## 适合谁

- **开发者**：想让 AI 助手更可靠地遵循项目规范（不乱建目录、不重复造轮子、知道项目有哪些工具可用）
- **团队**：想统一 AI 助手的行为模式（所有人用同一个框架，AI 表现一致）
- **AI 工具用户**：想让 AI 从"一次性问答"升级为"长期伙伴"

## 快速开始

```bash
# 1. 下载并解压
# 从 GitHub Release 下载 zip，解压到你想放的位置

# 2. 初始化
python artifacts/ix-init-cli/main.py init

# 3. 在 Claude Code 或 ZCode 中打开这个目录，开始协作
```

初始化时你可以设置：
- 助手的**昵称**和对你的**称呼**（个性化交互）
- **Git 模式**：纯本地版本库，还是推送到 GitHub

## 核心能力

| 能力 | 说明 |
|------|------|
| **长期记忆** | AI 启动时自动加载项目规范，不需要每次重复说明 |
| **能力发现** | AI 知道项目里有哪些工具可用，按意图自动匹配 |
| **Agent 编排** | 定义多步骤业务流程（拉数→分析→报告），AI 按流程执行 |
| **定时执行** | 注册到系统调度器（macOS/Windows），到点自动跑 |
| **一致性审计** | 自动检查文件结构与规范是否一致 |
| **基线更新** | 框架升级时，保留你的定制内容，只更新框架部分 |

## 设计理念

- **规则先行**：先定规则，AI 按规则行事，不靠"提醒"
- **可发现**：AI 不需要搜索就知道系统有什么能力
- **不发明**：AI 不自作主张创建目录、归档文件、增加步骤
- **跨平台**：macOS 和 Windows 都支持
